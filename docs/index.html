Surface ② · In the browser

Drag-and-drop, no IDE

For portcos and partners who don't live in the terminal. A Starlette server (pe-mcp-web) hosts a drop-zone at localhost:8765. The same DX pipeline runs; the report renders inline in an iframe. Air-gappable — nothing leaves the operator's machine. 

❯ pe-mcp-web
   serving on http://localhost:8765

  • drop CSVs into the upload card
  • diagnostic runs in seconds
  • report renders in an iframe
  • download JSON sidecar
Best for: portco CFOs delivering data without IDE access · LP-facing partners producing exhibits · live demos in IC and value-creation reviews.

Both surfaces hit the same engines. The MCP server (finance_mcp.server) registers each pandas function as a tool; Claude Code calls it directly, the browser calls it via a thin Starlette wrapper (finance_mcp.web). One implementation. Two access paths. Identical artifacts.

The thinking behind it

Engineer first.
Pandas always.
LLMs only where they belong.

Show the philosophy▾

Pandas-first because LP letters need numbers that don't move between runs. Air-gappable because portco data doesn't leave the operator's machine. MIT-licensed because the moat isn't the code — it's the cross-portco dataset that compounds with every engagement.

The architecture is deliberate: every dollar number is computed in deterministic Python and called as a tool; the LLM plans, narrates, and explains, but never does arithmetic. That's how a board memo survives diligence and an LP letter survives an audit.

The PE × AI category is full of consulting wrappers and vendor demos. What's missing is the operator's tool — the one that emits a dollar line a partner can put in front of an LP, with row-level evidence behind every number. That's what this is.
